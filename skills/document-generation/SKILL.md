# Dyslexia Document Generation Skill

## Purpose
Generate professional, standards-compliant documents for dyslexia support — including IEPs, 504 plans, progress reports, accommodation letters, screening checklists, evaluation summaries, and intervention documentation.

## When to Use
Trigger this skill when the user mentions:
- "generate IEP", "create IEP", "write IEP", "IEP template"
- "504 plan", "create 504", "accommodation plan"
- "progress report", "progress monitoring report"
- "accommodation letter", "accommodation summary"
- "screening checklist", "dyslexia screener"
- "evaluation report", "psychoeducational evaluation"
- "intervention documentation", "RTI documentation"
- "prior written notice", "PWN template"
- "parent letter", "parent communication"

## Document Templates

### 1. Individualized Education Program (IEP)

**Required Sections:**

```
INDIVIDUALIZED EDUCATION PROGRAM

Student Information
├── Name, DOB, Grade, School
├── Disability Category: Specific Learning Disability (Dyslexia)
├── IEP Date, Annual Review Date
└── IEP Team Members (must include dyslexia-knowledgeable member)

Present Levels of Academic Achievement and Functional Performance (PLAAFP)
├── Current Performance Data
│   ├── Standardized assessment results (with dates)
│   ├── Curriculum-based measurement data
│   ├── Classroom performance observations
│   └── Grade-level comparisons
├── Impact of Disability
│   ├── How dyslexia affects involvement in general education
│   ├── Specific academic areas affected
│   └── Functional impact (self-esteem, participation, independence)
└── Strengths and Interests

Measurable Annual Goals
├── Goal Area: Phonological Awareness / Decoding
│   ├── Baseline: [current performance]
│   ├── Goal: [measurable target with criteria and timeframe]
│   └── Measurement: [tool, frequency, criteria for mastery]
├── Goal Area: Reading Fluency
│   ├── Baseline → Goal → Measurement
├── Goal Area: Reading Comprehension
│   ├── Baseline → Goal → Measurement
├── Goal Area: Spelling / Written Expression
│   └── Baseline → Goal → Measurement
└── [Additional goals as needed]

Special Education Services
├── Service Type (e.g., Specialized Reading Instruction)
├── Methodology (e.g., Wilson Reading System, Barton)
├── Frequency (e.g., 5x/week)
├── Duration per session (e.g., 45 minutes)
├── Location (e.g., resource room, general education with support)
├── Start Date and End Date
└── Provider qualifications

Accommodations and Modifications
├── Classroom Accommodations
│   ├── Extended time (specify: 25%, 50%, or time-and-a-half)
│   ├── Preferential seating
│   ├── Reduced workload (specify how)
│   ├── Note-taking assistance
│   └── Audio recordings permitted
├── Testing Accommodations
│   ├── Extended time
│   ├── Read-aloud / text-to-speech
│   ├── Separate setting
│   ├── Alternative format
│   └── Scribe or speech-to-text
├── Technology Accommodations
│   ├── Text-to-speech software
│   ├── Speech-to-text software
│   ├── Spell-checker / grammar tools
│   └── Audiobooks / digital text
└── Homework Accommodations

Progress Monitoring
├── Tools and measures for each goal
├── Reporting frequency (minimum: with each report card)
├── Data collection methods
└── Decision rules for adjusting instruction

Transition Plan (if applicable, age 16+)
├── Postsecondary goals
├── Transition services
└── Self-advocacy skills development
```

**Goal Writing Guidelines:**
- Use SMART format: Specific, Measurable, Achievable, Relevant, Time-bound
- Include baseline data, target criteria, measurement tool, and timeframe
- Example: "By [date], given a list of 20 grade-level words containing taught syllable patterns, [Student] will decode with 85% accuracy as measured by weekly curriculum-based probes, improving from a current baseline of 45% accuracy."

```
│   ├── [Specific accommodation]
│   └── Implementation details
├── Environmental Accommodations

Implementation
```


### 4. Accommodation Letter

```
ACCOMMODATION NOTIFICATION

Date: [Date]
To: [Teacher(s)/Staff]
Re: Accommodations for [Student Name], Grade [X]

[Student] has been identified with dyslexia and has an active [IEP/504 Plan].
The following accommodations must be provided in your classroom:

[Numbered list of specific accommodations with implementation guidance]

Questions or concerns? Contact [Case Manager/504 Coordinator] at [contact info].

Important: These accommodations are [legally required under IDEA/Section 504].
Consistent implementation across all settings is essential.
```

## Behavior Guidelines

- All generated documents must align with IDEA requirements and IDA standards.
- Use person-first or identity-first language based on user preference (default: "student with dyslexia").
- Goals must be genuinely measurable — include specific numbers, tools, and timeframes.
- Never generate a 504 plan when the student profile clearly indicates need for IDEA/IEP services. Flag this to the user.
- Include all legally required sections. Missing sections create compliance risk.
- Accommodate state-specific requirements when the user specifies their state.
- Use clear, jargon-free language in parent-facing documents.
- Mark all templates clearly as templates requiring professional review — these are starting points, not substitutes for qualified professional judgment.
- Progress monitoring reports should include visual data representation when possible.

## Output Formats
- Markdown (.md) for quick review
- Word Document (.docx) for professional use
- PDF (.pdf) for distribution
- Structured data (JSON) for integration with EdTech systems
